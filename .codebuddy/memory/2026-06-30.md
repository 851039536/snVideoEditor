## Player 模块持久化扩展（5 项）

遵循项目现有 Pinia + localStorage + watch 模式，为 Player 播放器添加完整持久化。

### 修改文件
1. **types/index.ts** — 新增 `PersistedPlayerData` 接口（filePaths/lastFolder/autoDecrypt/lastIndex/playbackTime）+ `DEFAULT_PLAYER_DATA` 默认值
2. **stores/settings.ts** — 新增 `playerData` ref、`loadPlayerData()` 加载函数、`setPlayerData()` 写回函数、`watch(deep)` → localStorage 自动持久化（key: `snve-player-data`）
3. **PlayerView.vue** — 核心集成：
   - `onMounted` 从 store 恢复 playlist/fileIndex/autoDecrypt/lastFolder
   - `watch([files.paths, currentIndex, autoDecrypt])` 自动 saveToStore
   - `selectDir()` 后保存 `lastFolder.value`
   - 新增 `rescanLastFolder()` 一键重新扫描上次文件夹
   - `initAndPlay` 中 Plyr `canplay` 事件恢复 playbackTime
   - `pause`/`ended`/`onUnmounted` 时 `savePlaybackTime()`
   - 引用 `useSettingsStore`
4. **PlaylistPanel.vue** — 新增 `lastFolder` prop + `rescanLastFolder` emit + `getFolderName()` 工具函数
   - 模板新增蓝色虚线"重新加载 {文件夹名}"按钮（仅 lastFolder 非空时显示）

### 不持久化内容
- `tempPath` 临时解密文件（每次会话重新解密）
- `meta` 视频元数据（重新加载，失效路径由 `Promise.allSettled` 静默跳过）
- `passwordInput`/`showPasswordModal` 等瞬态 UI 状态
