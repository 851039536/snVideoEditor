## Player Plyr 三项扩展功能（预览缩略图、标记点、工具栏）

基于 Plyr 3.8.4 原生能力扩展三项功能，修改 7 个文件。

### 1. 预览缩略图 (Preview Thumbnails)
- **ffmpeg.ts**：新增 `generateThumbnailSprite()` 函数（~180 行），三步流程：
  1. FFmpeg fps 滤镜提取帧（例如每 5 秒一帧 160×90）
  2. concat + tile 滤镜拼接精灵图
  3. 生成 WebVTT 文件（#xywh 坐标定位每帧）
  4. `cleanupThumbnailDir()` 清理中间产物；`secondsToVttTime()` 时间格式转换
- **index.ts**：注册 `video:generateThumbnails` IPC handler（wrapOperation 模式，lockType: thumbnail）
- **preload/index.ts + index.d.ts**：新增 `generateThumbnails()` API + 类型；ProgressInfo.type 新增 `'thumbnail'`
- **PlayerView.vue**：
  - `hashPath()` 生成视频路径 hash 作为缓存 key
  - `generateThumbnailsIfNeeded()` 异步触发（playFile/decryptAndPlay 后 void 调用，不阻塞播放）
  - 自适应间隔：`Math.max(5, Math.ceil(dur / 100))` 约 100 张缩略图
  - 加密文件使用 tempPath（通过 getScreenshotInputPath 获取）
  - Plyr options 中传入 `previewThumbnails: { enabled, src: vttUrl }`

### 2. 进度条标记点 (Markers)
- **types/index.ts**：新增 `ScreenshotMarker { time, label }` 接口；PersistedPlayerData 新增 `screenshotMarkers` 字段
- **PlayerView.vue**：
  - `addMarker(timeSec)` 去重（±1 秒范围）+ 自动编号标签
  - `doSingleCapture` 和 `batchCapture` 成功时调用 addMarker
  - `saveToStore()` 持久化截图标记点；`onMounted` 恢复
  - Plyr `markers: { enabled, points }` 选项传入
- **_player.scss**：`.plyr__progress__marker` 圆点样式（accent-blue 色，hover 放大）

### 3. 视频下方工具栏
- **PlayerView.vue** 模板：video-player-wrapper 内新增 `.player-toolbar` 区域
  - 控件栏切换按钮（Eye/EyeOff 图标 + `toggleControlsOverlay`）
  - 播放速度快捷按钮组（6 档 0.5×-2×，当前速度高亮 + `setSpeed(speed)`）
  - 自动隐藏控件开关（`autoHideControls` watcher 动态控制 Plyr controlshidden 事件）
- **initAndPlay**：Plyr 配置 `hideControls: autoHideControls.value`；`ratechange` 事件同步 `currentSpeed`
- **watch(autoHideControls)**：动态注册/移除 `controlshidden` handler，确保关闭自动隐藏时控件始终可见
- **_player.scss**：`.player-toolbar { user-select: none }`

### 关键设计
- 缩略图生成异步非阻塞，失败静默降级（console.warn）
- 标记点使用 Plyr 原生 markers 渲染，避免自绘
- 工具栏状态通过 Pinia + localStorage 持久化（screenshotMarkers 字段）
- `@use "./_player"` scoped 导入标记点 + 工具栏样式
