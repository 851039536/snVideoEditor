## 修复 @/ 路径别名 ts-plugin 错误

- 所有 renderer .vue 文件都报 `找不到模块"@/..."` ts-plugin 错误
- 根因：tsconfig.web.json 缺少 `paths` 映射 + `include` 中 `src/preload/*.d.ts` + `composite: true` 冲突
- 修复 1：tsconfig.web.json 添加 `baseUrl: "."` + `paths: { "@/*": ["./src/renderer/src/*"] }`
- 修复 2：从 include 移除 `src/preload/*.d.ts`，改为在 env.d.ts 中添加 `/// <reference path="../../preload/index.d.ts" />` 引入预加载类型
- 修复 3：删除过期的 tsconfig.web.tsbuildinfo 和 tsconfig.node.tsbuildinfo 缓存
- 结果：CompressView.vue 的 4 个 @/ 路径错误全部消除

## 播放器 UI 重构与新功能（2026-06-29）

将 PlayerView.vue（782行单文件）拆分为 3 组件 + 1 SCSS + 1 types 文件：

### 新增文件
- `Player/types/index.ts`：PlayerEntry 接口、PlaybackSpeed 类型、PLAYBACK_SPEEDS 常量、formatSpeed() 格式化
- `Player/_player.scss`：共享 SCSS partial（进度条定制、控件覆盖层动画、缓冲预览、倍速菜单、拖拽排序、全屏覆盖层、响应式），所有颜色使用 CSS 变量
- `Player/PlaylistPanel.vue`：播放列表子组件，含 FileDropZone + 扫描文件夹 + 文件列表 + HTML5 拖拽排序（dragstart/dragover/drop）+ 悬浮删除 + 清空
- `Player/VideoControlsBar.vue`：控制条子组件，含自定义进度条（mousedown拖拽+缓冲预览）、音量滑块(0-1 range)、6档倍速菜单(0.5x-2.0x)、全屏按钮、播放/暂停/上/下曲

### PlayerView.vue 新增功能
- **音量滑块**：volume ref(0-1) + video.volume + onVolumeChange，滑块自动取消 mute
- **倍速播放**：playbackSpeed ref(0.5/0.75/1.0/1.25/1.5/2.0) + video.playbackRate
- **进度条拖拽**：mousedown→mousemove→mouseup 实时拖拽 + 缓冲预览（video.buffered TimeRanges）
- **全屏**：videoContainer.requestFullscreen() + document.exitFullscreen() + fullscreenchange 事件监听
- **居中播放覆盖层**：暂停时显示 56px 半透明圆环大播放按钮，点击视频画面播放/暂停
- **控制条自动隐藏**：播放时 3s 无鼠标移动自动淡出，暂停或鼠标移动时显示
- **拖拽排序**：handleReorder() 处理 from→to splice 移动，同步更新 currentIndex

### 技术要点
- 无 IPC/主进程变更，所有新功能纯前端
- 样式全部提取至 _player.scss，PlayerView.vue `<style>` 仅一行 `@use "./_player"`
- 保留所有现有业务逻辑（加密解密、自动解密、密码弹窗、视频信息卡片）
