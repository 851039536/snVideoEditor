### 修复 SplitMerge 裁剪合并 bug
- **根因1**：切到合并模式时 `files` 数组未清空，原始视频混入合并输入 → `watch(mode)` 新增 `merge` 分支清空 `files.value = []`
- **根因2**：ffmpeg `-ss` 在 `-i` 之后导致 `-c copy` 模式下裁剪不准 → 调整参数顺序为 `-ss startTime -i input -t duration`
- 修改文件：`views/SplitMerge/SplitMergeView.vue`、`main/modules/ffmpeg.ts`

### 5 个逻辑漏洞修复
1. **onProgress 内存泄漏**：preload `onProgress()` 内先 `removeAllListeners` 再注册；CompressView/EncryptView 补充 `onUnmounted(() => removeProgressListener())`
2. **elapsed 不实时**：`progress.ts` 中 elapsed 从 computed 改为 `ref + setInterval` 每秒自增，`finish()/reset()` 中 `clearInterval`
3. **videoMeta 竞态**：`loadVideoMeta()` await 返回后检查 `files[0]` 是否仍为请求时的 filePath，不匹配则丢弃
4. **模式切换残留**：`watch(mode)` 切 merge 时追加清空 `outputName`/`outputDir`/`errorMsg`
5. **temp 文件泄漏**：新增 `file:delete` IPC + preload `deleteFile` API；`removeClip` 时删 temp、merge 成功后遍历删已选 clip 的 temp
- 修改 7 个文件：`preload/index.ts`、`preload/index.d.ts`、`main/index.ts`、`stores/progress.ts`、`views/SplitMerge/SplitMergeView.vue`、`views/Compress/CompressView.vue`、`views/Encrypt/EncryptView.vue`
