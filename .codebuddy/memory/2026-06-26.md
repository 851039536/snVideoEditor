## FFmpeg 路径解析失败修复（spawn UNKNOWN）
- 根因：node_modules/ffmpeg-static/ffmpeg.exe 被 Windows 安全策略阻止（Access is denied），但系统 PATH 中有可用的 ffmpeg（E:\Program\ffmpeg-8.0.1-essentials_build\bin\）
- 新增 resolveFfmpegPath() / resolveFfprobePath() 多级回退：
  1. 环境变量（FFMPEG_PATH / FFPROBE_PATH）
  2. ffmpeg-static / ffprobe-static require（加 canExecute() 校验）
  3. __dirname 逐级回退搜索 node_modules（加 canExecute() 校验）
  4. 系统 PATH 查找（where ffmpeg / which ffmpeg + canExecute() 校验）
- 新增 canExecute() 函数：在 Windows 上用 spawnSync -version 验证二进制是否可运行
- 错误消息增强：spawn error 时显示具体路径路径

## FFmpeg 裁剪时长不准修复（-c copy 关键帧问题）
- 问题：选 2 秒实际切出十几秒
- 根因：`-ss` 在 `-i` 之前 + `-c copy` = 输入跳转模式，只能跳到最近关键帧（GOP 边界），H.264 关键帧间隔 2-10 秒
- 修复：args 顺序改为 `-i INPUT -ss START -t DURATION -c copy`（-ss 在 -i 之后）
- 效果：输出时长精确，但可能从下一个关键帧开始（偏移最多一个 GOP）。如需帧级精确需改为重新编码模式
